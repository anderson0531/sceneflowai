# SceneFlow Real Estate Service v1 Playbook

## 1) Service Model and Positioning

### Service promise
SceneFlow Real Estate Service v1 helps agents, brokerages, and real estate media teams produce listing walkthrough videos faster and with lower production overhead by combining:
- structured intake,
- AI-assisted generation,
- human QA review,
- multilingual delivery.

### Market positioning
- **Traditional model:** multi-role production, longer lead time, higher fixed costs.
- **SceneFlow model:** standardized workflow, lower barrier to launch, faster iteration.

### Primary buyer profiles
- Independent agents with limited production resources.
- Multi-agent brokerages needing consistent listing media output.
- Regional teams serving international buyers requiring multilingual content.

### Core value pillars
- Lower cost barrier per listing.
- Faster turnaround from intake to publish-ready cut.
- Agent/brand consistency via reference-driven workflows.
- Global-ready multilingual variants.

## 2) Package Matrix (Global-Ready)

### Package A — Listing Quick Tour
- **Use case:** fast publish for single property.
- **Runtime target:** 60-90 seconds.
- **Languages included:** 1.
- **Inputs:** up to 20 images, basic property facts, agent headshot (optional).
- **Outputs:** 1 final video, 1 thumbnail, 1 short caption draft.
- **Turnaround SLA:** 3 business days.
- **Revision policy:** 1 minor revision round.

### Package B — Guided Property Tour
- **Use case:** premium walkthrough with hosted narrative.
- **Runtime target:** 2-3 minutes.
- **Languages included:** 1.
- **Inputs:** up to 40 images, room-by-room notes, agent image, voice profile (or stock voice).
- **Outputs:** 1 final video, 2 cutdowns (15-30 sec), title/description drafts.
- **Turnaround SLA:** 5 business days.
- **Revision policy:** 2 revision rounds (1 structural, 1 polish).

### Package C — Global Buyer Pack
- **Use case:** multilingual listing distribution.
- **Runtime target:** 2-4 minutes.
- **Languages included:** 3 (base package), expandable via add-ons.
- **Inputs:** full listing asset set, approved base script, localization targets.
- **Outputs:** master language cut + localized variants + channel-ready metadata pack.
- **Turnaround SLA:** 7 business days initial, 2 business days per added language batch.
- **Revision policy:** 2 global rounds + 1 localization correction pass.

### Standard add-ons
- Extra languages (per language or language bundle).
- Rush delivery (24-48h tiered surcharge).
- Premium voice clone setup.
- Agent avatar styling/wardrobe direction.
- Listing refresh update (price/status/media refresh).
- Vertical social package (9:16 exports + cutdowns).

## 3) Pricing and Margin Framework

### Pricing formula (operational)
`Final Price = Base Package Fee + Add-Ons + Localization Multiplier + Rush Multiplier + Overage`

### Recommended controls
- **Base package fee:** fixed for A/B/C.
- **Localization multiplier:** per added language after included set.
- **Rush multiplier:** percentage premium by SLA compression.
- **Overage policy:** image count/runtime/revision over package limits.

### Suggested margin targets
- **Gross margin target:** 65-75% on standard SLA work.
- **Contribution margin floor:** 50% on rush work.
- **Localization margin:** 70%+ once workflows are templated.

### Scope guardrails
- Hard cap on included revisions.
- Explicit runtime and asset ceilings by package.
- Change requests after script sign-off treated as re-scope.

## 4) Sales Workflow and Assets

### Pipeline stages
1. Lead capture
2. Qualification
3. Discovery call
4. Solution fit + package recommendation
5. Proposal and quote
6. Close and deposit
7. Onboarding kickoff

### Qualification checklist (pass/fail)
- Listing status and market timeline defined.
- Required media assets available or feasible within 72h.
- Decision-maker identified.
- Budget range aligns with package tier.
- Target channels and language requirements known.

### Discovery call script (short form)
- What is the property and target buyer profile?
- What timeline are you trying to hit?
- Do you need multilingual versions?
- What currently blocks your listing video output?
- What level of agent presence (on-screen voice/avatar) is required?

### Proposal structure
- Client objective summary.
- Recommended package and rationale.
- Delivery timeline and milestone calendar.
- Revision scope and acceptance criteria.
- Compliance/disclosure requirements.
- Commercial terms and payment schedule.

### Onboarding script
- Confirm scope, timeline, and dependencies.
- Confirm asset checklist and content lock date.
- Confirm approval contacts and SLA response expectations.
- Confirm language/voice/legal disclosures before generation starts.

## 5) Client Intake Schema (Mandatory)

### Property metadata
- Listing address (or internal property ID).
- Property type, square footage, bed/bath count.
- Key selling points and must-mention features.
- Claims requiring exact wording (HOA, school, zoning, etc.).

### Asset requirements
- Room/location image set (recommended min/max by package).
- Exterior/context shots (street, amenities, neighborhood highlights).
- Agent/realtor photo(s) if avatar usage enabled.
- Voice sample or selected voice profile.
- Brand assets (logo, color preference, disclaimers).

### Delivery requirements
- Target platforms (MLS, YouTube, Instagram, WhatsApp, etc.).
- Aspect ratios needed (16:9, 9:16, 1:1).
- Language targets and priority order.
- CTA destination (phone, landing page, booking link).

### Compliance capture
- Likeness and voice consent confirmation.
- Region-specific disclosure requirements.
- Property claim validation acknowledgment.

## 6) Production SOP (Step-by-Step)

### Stage 0 — Intake validation (Ops Coordinator)
- Validate required fields and asset completeness.
- Flag missing items and block kickoff until complete.
- Assign package and production owner.

### Stage 1 — Blueprint and script draft (Producer)
- Build room-by-room story arc.
- Generate first script draft using package template.
- Prepare voice and language plan.

### Stage 2 — Client script approval (Client + Producer)
- Collect edits in one consolidated pass.
- Lock script baseline before production generation.

### Stage 3 — Scene production (Generation Operator)
- Generate room scenes from approved references.
- Apply agent avatar/wardrobe/voice settings.
- Produce base language master cut.

### Stage 4 — Localization and variant generation (Localization Operator)
- Generate translated scripts and voice tracks.
- Run timing/alignment checks by language.
- Produce localized cuts and export set.

### Stage 5 — QA and compliance review (QA Reviewer)
- Run visual, audio, factual, and disclosure checks.
- Validate package acceptance criteria.

### Stage 6 — Delivery and handoff (Account Manager)
- Deliver approved exports and metadata bundle.
- Collect approval and close project.
- Log feedback for template/system improvement.

## 7) Global Compliance and Disclosure Framework

### Required controls
- Written consent for use of uploaded human images and voice cloning.
- Confirmation that marketing claims are client-provided and verified.
- AI-assisted media disclosure policy per region/channel.

### Standard disclosure baseline
- Include configurable disclosure line:
  - “This video includes AI-assisted production elements.”
- Keep disclosure toggles configurable by jurisdiction and platform.

### Real estate claim policy
- No unverifiable claims in narration.
- Any regulated claims (schools, compliance statements, legal descriptors) must be client-approved verbatim.
- Track source field for each high-risk claim in intake.

### Region readiness checklist
- Country/region toggle in project intake.
- Region-specific legal checklist attachment.
- Legal review required for new region launch before sales activation.

## 8) QA Checklist and Approval Gates

### Gate A — Script QA
- Story flow follows room order and objective.
- No unsupported claims.
- Tone aligns to property tier and target audience.

### Gate B — Visual QA
- Agent likeness consistency.
- Room/location continuity and realism.
- Branding and wardrobe conformity.

### Gate C — Audio QA
- Voice clarity and pacing.
- Pronunciation accuracy (especially localized names/places).
- Mix balance and no clipping.

### Gate D — Localization QA
- Meaning fidelity to source script.
- Timing alignment to scene flow.
- Region-appropriate terminology.

### Gate E — Delivery QA
- Correct format/aspect ratio/output package.
- Metadata and CTA links validated.
- Final compliance/disclosure check passed.

## 9) Automation Roadmap (ROI-Prioritized)

### Phase 0 (Immediate, low complexity)
- Structured intake form with hard validation.
- Automated project folder/file naming conventions.
- Package-based template assignment.

### Phase 1 (Near-term, medium complexity)
- Auto-generate initial script from intake data.
- Auto-segment room sequences from uploaded asset groups.
- Automated multilingual draft generation workflow.

### Phase 2 (Scale, higher complexity)
- Batch render orchestration by package + language.
- QA assistant checks (missing rooms, duration drift, disclosure presence).
- Ops dashboard with SLA risk alerts and margin telemetry.

## 10) KPI Dashboard and Weekly Cadence

### Core KPI set
- Lead-to-close rate.
- Avg. turnaround time (intake complete -> delivery).
- First-pass approval rate.
- Revision rounds per project.
- On-time delivery percentage.
- Gross margin by package.
- Localization defect rate.
- Throughput per producer-equivalent.

### Weekly operating rhythm
- **Monday:** pipeline + capacity review.
- **Wednesday:** delivery risk and QA defect review.
- **Friday:** margin + process improvement retro.

### Alert thresholds
- First-pass approval < 70%.
- Avg revisions > 2.0 on package A/B.
- On-time delivery < 90%.
- Margin drop > 10 points week-over-week.

## 11) Team Structure and Capacity Model

### Initial team (lean launch)
- 1 Sales/Account Lead
- 1 Producer
- 1 Generation/Localization Operator
- 1 QA/Compliance Reviewer (can be part-time shared role)

### Capacity baseline (starting assumption)
- 8-12 Package A equivalents per week, or
- 5-7 mixed A/B projects per week, or
- 3-4 Package C projects per week.

### Hiring triggers
- Add Generation Operator when utilization > 80% for 3 consecutive weeks.
- Add QA Reviewer when defect rate or revision loops trend upward.
- Add Account Manager when active client count exceeds onboarding SLA.

## 12) 30/60/90-Day Rollout

### Day 0-30 (Launch foundation)
- Finalize package definitions and pricing.
- Deploy intake schema and SOP.
- Run first 5-10 pilot projects.
- Capture baseline KPI metrics.

### Day 31-60 (Stabilize and optimize)
- Standardize scripts and scene templates by property type.
- Introduce phase-1 automation (script draft + segmentation).
- Tighten QA gates from pilot defect data.
- Publish repeatable case-study outputs.

### Day 61-90 (Scale and expand)
- Launch multilingual package upsells at scale.
- Deploy batch render queue and SLA risk dashboard.
- Expand channel partnerships (brokerages/agency partners).
- Prepare region-by-region legal checklist rollouts.

---

## 13) Operational Governance Rules

- No production start without complete intake and signed consent.
- No script-to-production handoff without script lock.
- No delivery without all five QA gates passed.
- Scope changes after lock require formal change order.
- Every project retro feeds template and automation improvements.
